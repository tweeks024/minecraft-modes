# securityguard texture & model tools

The Security Guard's appearance is authored in [Blockbench](https://www.blockbench.net/),
a free 3D model and texture editor for Minecraft. This directory holds the
canonical Blockbench project file:

```
security_guard.bbmodel
```

## Workflow

1. Install Blockbench from <https://www.blockbench.net/>.
2. Open `security_guard.bbmodel` (double-click, or **File → Open** in Blockbench).
3. The 3D viewport shows the guard with the current texture applied. Edit:
   - **Texture** — switch to the *Paint* tab, paint pixels in 2D or 3D.
   - **Cubes** — switch to the *Edit* tab, drag cubes, resize, add new ones.
4. Save the bbmodel: **File → Save** (overwrites `security_guard.bbmodel`).
5. Export the texture back into the resource pack:
   - **File → Export → Export Texture (PNG)**
   - Overwrite `../src/main/resources/assets/securityguard/textures/entity/security_guard.png`
6. If you changed cube geometry (added a part, moved a part, resized), update
   `SecurityGuardModel.java` to match. Blockbench coords map to Java coords as:
   - Blockbench `from`/`to` → Java `addBox(x, y, z, w, h, d)`
   - Blockbench `origin` → `PartPose.offset(x, y, z)`
   - Java `y` is inverted: `Java_y = 24 - Blockbench_y` for head children.
7. `./gradlew :securityguard:build && ./gradlew :securityguard:runClient` to verify.

## Why Blockbench?

- Visual feedback on what UV slot maps to which face of which cube.
- Faster iteration than hand-editing pixel coordinates in code.
- Standard tool for Minecraft model authoring — the bbmodel format is portable
  and well-supported.

## Notes

- The bbmodel embeds the current PNG as base64 at creation time so the file is
  self-contained. Once Blockbench saves the texture, the `source` field is
  refreshed to match the latest pixels.
- The bbmodel file is NOT regenerated by any script. It is the source of truth.
  If it gets corrupted, recover it from git history.
- The **hat overlay** (the auto-generated 8×8×8 inflated head copy that
  HumanoidModel adds by default) is disabled in `SecurityGuardModel.java` —
  our explicit `cap_brim` and `cap_crown` cubes replace it. If you re-enable
  the hat overlay, paint its UV region (texture coords 32..64, 0..16)
  transparent or it will hide the face features painted at HEAD_FRONT.
